# AI Detection Page Audit

Audit date: 2026-07-16

## Root cause

The previous Model Feature Explanation chart was not an XAI/model explanation. `dashboard.py` created a fixed dictionary from display values and plotted it directly. In particular, it used raw Wi-Fi channel frequency (for example `5180000`) as a chart value, while other values were synthetic flags. This caused the million-scale bar and zero/near-zero bars.

There was no persisted `scan_history.csv`; the risk trend was based on the current in-memory scan and therefore disappeared after reload or had too few points to render.

## Fixes applied

- Added model-derived `feature_contributions` to `HybridEnsembleDetector.predict`.
  - Uses the Random Forest's learned `feature_importances_` and the live prediction input vector.
  - Converts inputs to unitless magnitudes before visualization, preventing Hz channel values from dominating.
  - Removes NaN/infinite values, clamps invalid values, normalizes contributions to 100%, and emits percentage values.
- Removed all hardcoded XAI values from `dashboard.py`.
- Replaced truncated labels with full readable labels and horizontal Altair bars with tooltips.
- Added top contribution percentage and an explanation summary for each selected network.
- Corrected the model path to avoid transforming inputs with an unfitted scaler.
- Persisted `scan_history.csv` with `timestamp`, `ssid`, `threat_score`, and `combined_risk` on every completed scan.
- Built the trend from `scan_history.csv`, with useful empty/insufficient-data messages.
- Added safeguards for one-network threat comparisons and distributions.
- Made the dashboard reload `current_scan.csv` when in-memory scanner results are unavailable, so every network in the latest completed scan remains available in Explain Network after a dashboard refresh.

## Files modified

- `ml_ensemble.py`
- `background_scanner.py`
- `dashboard.py`
- `xai_explain.py`
- `scan_history.csv` (created by the validated scan flow)

## Validation

- A demo scan completed with three networks.
- `scan_history.csv` was created with three history rows.
- Every available model explanation had finite values and normalized contribution total between 99.9% and 100.1%.
- Python syntax compilation passed for all changed modules.

## Remaining issues

- SHAP is not installed or used by this project. The implementation uses intrinsic Random Forest feature importances combined with the exact current prediction vector, which is reliable and dependency-free but is not a SHAP value calculation.
- Serialized scikit-learn models were generated by version 1.8 while the active environment uses 1.9. Retrain or pin the model environment before production use.
- If model artifacts cannot load, the scanner marks model predictions/XAI as unavailable instead of fabricating explanations.
